Pattern Classification: Algorithm Testing

The algorithms for testing of the pattern classification operations are a powerful tool for the classification process. There are two main algorithms for testing according to the selected parameters related to the distance and similarity metrics, data normalization, and the classification algorithms.

The Train/Test function uses the feature vectors defined in the Train set and apply to those the normalization method specified. The same parameters used in the normalization of the Train set are then applied to the Test set and according to the Distance/Similarity metric selected and the Classification Algorithm, a class is assigned to each feature vector in the Test set.

The Leave One Out function uses the feature vectors defined in the Feature file and creates a Test set with one of the elements in the vector and uses the remaining feature vectors as the Train Set. Then the Train/Test set classification is applied to the current feature vector and the class is assigned to this. The next element in the feature vector is as Test set and the classification continues until all the elments in the Feature file has been used as Test vector. This is common used when the database or image set are not to much large to create good Test and Train sets of features.

  • train_test_cvip - reads test and training files of feature vectors and performs a classification using the specified normalization method, distance/similarity measure and classification algorithm
  • leave_one_out_cvip - reads a feature vectors and creates train/test feature vector sets to perform a classification using the specified normalization method, distance/similarity measure and classification algorithm
